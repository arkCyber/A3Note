# A3Note 浏览器开发模式

## 概述

A3Note 支持两种运行模式：

1. **Tauri 桌面模式** - 完整的桌面应用，可访问本地文件系统
2. **浏览器开发模式** - 使用模拟数据的 Web 版本，用于快速开发和测试

## 浏览器模式特性

### 自动检测
应用会自动检测运行环境：
```typescript
const isTauri = typeof window !== 'undefined' && '__TAURI__' in window;
```

### 模拟功能

#### 1. 文件系统操作
浏览器模式提供完整的文件系统模拟：

- ✅ **读取文件** - 从内存中的模拟数据读取
- ✅ **写入文件** - 保存到内存（会话期间保持）
- ✅ **创建文件** - 添加到模拟文件列表
- ✅ **删除文件** - 从模拟列表中移除
- ✅ **列出目录** - 返回预定义的文件树
- ✅ **搜索文件** - 在模拟内容中搜索

#### 2. 对话框
- ✅ **打开工作区** - 自动返回 `/demo` 路径
- ✅ **打开文件** - 自动返回 `/demo/README.md`

#### 3. 预置演示数据

**文件结构**:
```
/demo
  ├── README.md
  └── notes/
      ├── note1.md
      └── note2.md
```

**文件内容**:
- `README.md`: 欢迎页面
- `note1.md`: 示例笔记 1
- `note2.md`: 示例笔记 2

## 启动浏览器模式

### 方法 1: 仅前端开发
```bash
npm run dev
```
然后访问 http://localhost:1420

### 方法 2: 完整 Tauri 开发
```bash
npm run tauri:dev
```
这会同时启动前端和 Tauri 后端

## 限制和注意事项

### 浏览器模式限制

1. **数据不持久化**
   - 刷新页面会丢失所有更改
   - 无法访问真实文件系统

2. **对话框模拟**
   - 无法选择真实目录
   - 无法浏览本地文件

3. **功能限制**
   - 某些 Tauri 特定功能不可用
   - 系统集成功能受限

### 何时使用浏览器模式

✅ **适合**:
- UI/UX 开发和调整
- 组件样式调试
- 快速原型验证
- 插件系统测试
- 主题和国际化测试
- 响应式设计测试

❌ **不适合**:
- 文件系统功能测试
- 性能基准测试
- 生产环境部署
- 真实数据操作

## 故障排除

### 问题: 应用无法启动

**解决方案**:
```bash
# 清理并重新安装依赖
rm -rf node_modules
npm install

# 重新启动开发服务器
npm run dev
```

### 问题: 端口 1420 被占用

**解决方案**:
```bash
# 查找占用端口的进程
lsof -ti:1420

# 终止进程
kill -9 $(lsof -ti:1420)

# 重新启动
npm run dev
```

### 问题: 热重载不工作

**解决方案**:
1. 检查 `vite.config.ts` 配置
2. 清理浏览器缓存
3. 重启开发服务器

## 性能优化

### 浏览器模式优化

1. **减少模拟数据大小**
   - 只包含必要的演示文件
   - 避免大量模拟内容

2. **使用懒加载**
   - Tauri API 动态导入
   - 按需加载组件

3. **缓存策略**
   - 利用浏览器缓存
   - 优化资源加载

## 部署说明

### 浏览器模式部署

浏览器模式可以部署为静态网站：

```bash
# 构建生产版本
npm run build

# 预览构建结果
npm run preview

# 部署到静态托管服务
# - Netlify
# - Vercel
# - GitHub Pages
```

**注意**: 部署的浏览器版本将使用模拟数据，适合演示和预览。

### Tauri 桌面应用部署

```bash
# 构建桌面应用
npm run tauri:build

# 生成的安装包位于:
# src-tauri/target/release/bundle/
```
